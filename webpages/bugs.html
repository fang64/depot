Known Bugs

Last Update: December 20, 1994

This is a list of known defects with depot and dpp. The first part of each item is the defect tracking number.

Feature requests are not included in this list.

Depot

depot.pref Issues

  • 940802-003 - If the ignore statement in the depot.pref is given items separated by white space rather than commas, depot will go into an infinite loop.

  • 940831-003 - the target.owner/target.group depot.pref options do not set the directory. They only set files within a directory. There is no known workaround.

  • 940831-003 - the target.owner/target.group depot.pref options do not set the directory. They only set files within a directory. There is no known workaround.

  • 950220-001 - collection.installmethod will work only with copy or link.

    Runtime Issues

  • 941130-002 - If the modes on a copied file changes but the modification time doesn, the modes are not updated on the files. The workaround is to 'touch' the changed file, thereby changing the modification time.

  • 940518-017 - There is a core leak in the LEX/YACC code.

  • 940518-015 - The user is not warned if there are duplicate preference options (which may result in weird/unexpected behavior).

  • 950303-001 - Invalid hints genereate odd to bad behavior with non-intuitive error messages.

    Dpp

  • 941010-002 - the ~deletecommand depot.conf option is not properly processed. If included in the depot.conf.proto, it will be silently discarded and never make it into the final depot.conf. There is no workaround.

  • 950303-002 - searchpath <collection> doesn't work right
