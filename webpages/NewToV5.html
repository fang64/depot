Changes From Depot Version 4 to Version 5

  1. Files renamed
      struct.depot -> depot.db
      custom.depot -> depot.pref

  2. Files added
      depot.conf - an environment wide depot.conf (this exists in the depot sub-directory)

  3. Defaults changed
    • The default targetdir is now '.' as opposed to '..'. Specifically, this means that running depot in the depot subdirectory no longer works.
    • A depot.pref file must exist. Previously depot could run without a custom.depot file being present.

  4. Quick mode is quicker (but less consistent)
      If -Q is used, then it does not check the target directory. This means that (1) unmapped files aren't removed and (2) invalid links may result in depot hanging.

  5. The command syntax has changed. See the "Using Depot" for details.

  6. Filters
      When installing files from a collection, it used to be that you could either copy or link the file. Now, you can pass the file through a "filter." For example, you could store all the files in a collection as compressed files and the "filter" used to install the files would be uncompress. See "Using Depot" for details.

  7. Command line
      Collections to be updated are no longer specified on the command line or via the -f switch. All collections specified by the 'path' or 'searchpath' preference option will be processed.
